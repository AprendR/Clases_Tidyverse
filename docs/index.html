Gramtica de manejo de datos

Este paquete permite el manejo de datos usando una gramtica con verbos consistentes y que permiten resolver de forma clara y concisa casi cualquier procesamiento de datos.

Entre algunas de sus ventajas tenemos:

  • Sintaxis interpretable
  • Es extensible para usar con bases de datos, spark o data.table
  • Compatible con el resto de paquetes que forman parte de tidyverseEs una coleccin de paquetes que permiten el manejo de datos manteniendo un gramtica en comn

Pipe

Una caracterstica que se distintiva de esta gramtica de procesamiento de datos es el uso de una pipe que se define como el smbolo %>%.

Este smbolo permite encadenar varias instrucciones o verbos que se ejecutaran uno tras otro.

En este encadenamiento se puede obviar el paso del data.frame que se esta procesando, considerandolo como el primer argumento de cada verbo.

dplyr

Encadenando comandos

Una de las ventajas de esta gramtica es que estos comandos se pueden usar de forma encadenada uno tras el otro.

Esto se puede hacer mediante el operador pipe

situacion_laboral_madre %>%
  filter(Ao == "2013*") %>% 
  mutate(ao_entero = as.integer(substr(Ao, 1, 4))) %>% 
  select(ao_entero, Estudiante, Otra)
## # A tibble: 1 x 3
##   ao_entero Estudiante  Otra
##        <int>      <dbl> <dbl>
## 1       2013       7.70  15.7
